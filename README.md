
# （1）制作サイトのタイトル：
Near Café

# （2）制作サイトの説明（40文字程度）：
土日等カフェで作業することが多いですが、空いているカフェを見つけるにも時間がかかります。<br>
現在地近くのカフェの空き状況・Wi-Fi/電源有無が一元的に見られるサービスがあったらいいなと思い作成しました。<br>
<br>
https://renren-geek.github.io/near-cafe-01/

# （3）工夫した点・こだわった点：
## Service Page
・検索箇所にこだわりました(かなり時間かかりました)<br>
・Google Mapから取れるデータはそのまま表示（例：お店の名前、場所、写真、Googleの評価、営業時間の現在状態）<br>
・Google Mapから取れないデータは仮置きして表示（例：リアルタイムの空席／混雑率、Wi-Fi /電源の有無）<br>
・また、上記はGoogle Cloud Consoleのアカウントを作成(無料トライアル)して、APIを取得する必要があったので、それも対応しました。<br>

## その他 Page
・基本的には①UXを高める為に必要な機能、②自分が作ってみたい機能を作りました。<br>
・①全頁から「Service」のページに飛べる、スクロールしてもヘッダーが固定される等<br>
・②ハンバーガーメニューの作成、About-Us Pageの画像をスライド形式にする（画像枚数は4枚、20秒で4分割(1画像5秒表示)、1枚目0秒後に再生、2枚目5秒後に再生、、、）<br>

# （4）難しかった点・次回トライしたこと（または機能）
・Google Map上で取れないデータをどうやって仮置きするか、が難しかったです。<br>
・次回は仮置きした部分を本実装してみたいです（混雑状況：お店側にAPI連携を求める、リアルタイムで口コミを拾ってくる、電源/Wi-Fi：店舗が申告しているデータ、口コミなどから拾ってくる等）
